| 項目    | 内容                                              |
| ----- | ----------------------------------------------- |
| アプリ名  | **LostAI**                                      |
| ジャンル  | AI対話型・記憶復元推理ゲーム                                 |
| テーマ   | 記憶、喪失、復元、AIの過去                                  |
| プレイ時間 | 1プレイ 3〜5分                                       |

# 2. コンセプト

## 2.1 一言コンセプト

> **記憶を失ったAIに5回だけ質問し、断片的な思い出から「何のAIだったのか」を推理するWebゲーム。**

---

## 2.2 バックストーリー

プレイヤーは、古いAI管理端末を操作している。

ある日、AIのチャットログを整理していたところ、誤ってAIの記憶データをすべて削除してしまう。

しかし、記憶は完全に消えたわけではない。
深層記憶領域には、わずかな断片が残っている。

ただし、記憶領域は崩壊寸前。
AIに話しかけて記憶を呼び出せるのは **残り5回だけ**。

プレイヤーは5回の会話から記憶の断片を集め、
このAIがかつて何のために作られたAIだったのかを推理する。

---

# 3. ゲームの基本ルール

## 3.1 基本フロー

```text
タイトル画面
↓
バックストーリー / ルール説明
↓
ゲーム開始
↓
AIの初期記憶を表示
↓
プレイヤーが質問する
↓
AIが断片的に回答する
↓
最大5回まで繰り返す
↓
最終回答画面
↓
プレイヤーが「何のAIだったか」を回答
↓
AIが採点
↓
正解なら正解動画
↓
不正解なら不正解動画
↓
総合評価画面
↓
タイトル画面へ戻る
```

---

## 3.2 質問ルール

| ルール   | 内容               |
| ----- | ---------------- |
| 質問回数  | 5回まで             |
| 入力形式  | 自由入力             |
| 1回の質問 | 1メッセージ送信ごとに1回消費  |
| 雑談    | 雑談でも1回消費         |
| 直接質問  | AIは直接答えない        |
| 最終回答  | 5回終了後、または途中で推理可能 |
| ゲーム終了 | 最終回答後は追加質問不可     |

---

## 3.3 AI回答ルール

AIはChatGPT APIを使って回答する。

### 必須条件

* 正体を直接言わない
* 直接的な質問には答えない
* 聞かれたことのみ答える
* 余計なヒントを出しすぎない
* 記憶が断片的に戻るような口調にする
* 回答は短めにする
* 5回で推理できる程度の情報量にする

---

## 3.4 直接質問の扱い

### 直接質問の例

```text
あなたは駅案内AIですか？
あなたは病院受付AI？
正体は何？
何のAIだったの？
答えを教えて
正解は？
```

### AIの返答方針

直接的な正否は返さない。

例：

```text
その名前を直接思い出すことはできません。
でも、人の流れを見つめていた記憶だけは残っています。
```

または、

```text
正体そのものは、まだノイズの奥にあります。
ただ、白い光と、待っている人々の気配を覚えています。
```

# 4. 勝敗条件

## 4.1 正解条件

プレイヤーの最終回答が、正体に意味的に近い場合は正解とする。

完全一致でなくてもよい。

### 例

| 正解         | プレイヤー回答       | 判定 |
| ---------- | ------------- | -- |
| 病院受付AI     | 病院で患者を案内するAI  | 正解 |
| 災害救助ドローンAI | 災害現場で人を探すドローン | 正解 |
| 駅案内ロボット    | 駅で迷子を案内するAI   | 正解 |
| 図書館検索AI    | 本を探す図書館のAI    | 正解 |

---

## 4.2 不正解条件

正体と大きく異なる場合は不正解とする。

### 例

| 正解         | プレイヤー回答 | 判定  |
| ---------- | ------- | --- |
| 病院受付AI     | 掃除ロボット  | 不正解 |
| 災害救助ドローンAI | カフェ注文AI | 不正解 |
| 駅案内ロボット    | 宇宙船管理AI | 不正解 |

---

## 4.3 採点基準

| 項目     | 内容                     |
| ------ | ---------------------- |
| 一致率    | 正解との意味的な近さ             |
| 質問力    | どれだけ有効な質問をしたか          |
| 不要質問   | 推理にあまり役立たなかった質問        |
| 良かった言葉 | AIの記憶復元に有効だった質問や単語     |
| 記憶回復率  | 会話全体からどれだけAIの記憶が復元されたか |
| 総合ランク  | S / A / B / C / D      |
| その後のAI | 成功・失敗後の短い後日談           |

# 6. 総合評価画面

## 6.1 目的

単なる正解・不正解で終わらせず、プレイヤーの会話をAIが振り返る。
ここで「AIを使ったゲームらしさ」を強く出す。

---

## 6.2 表示内容

| 項目       | 内容                |
| -------- | ----------------- |
| 正体       | AIが本当は何だったか       |
| プレイヤーの回答 | 最終回答              |
| 一致率      | 0〜100%            |
| 記憶回復率    | 0〜100%            |
| 総合ランク    | S / A / B / C / D |
| 良かった質問   | 推理に有効だった質問        |
| 良かった言葉   | 記憶復元に効いた単語        |
| 不要だった質問  | 推理にあまり貢献しなかった質問   |
| 改善アドバイス  | 次回どう聞けばよかったか      |
| その後のAI   | 成功・失敗後の短いストーリー    |
| ボタン      | タイトルへ戻る           |

---

## 6.3 正解時の総合評価例

```text
総合評価：A

正体：
災害救助ドローンAI

あなたの回答：
災害現場で人を探すドローンAI

一致率：96%
記憶回復率：91%

良かった質問：
「あなたは人を助けていましたか？」
「あなたには体がありましたか？」

良かった言葉：
人を助ける、探す、空中、危険な場所

必要なかった質問：
「あなたは毎日同じ仕事をしていましたか？」
この質問は少し抽象的で、正体の特定にはつながりにくい内容でした。

改善アドバイス：
移動方法や、誰を対象にしていたかを早めに聞くと、より正確に推理できました。

その後のAI：
記憶を取り戻したLostAIは、再び救助用ドローンとして起動した。
今度は失われた誰かを探すのではなく、自分を思い出させてくれたあなたの記録を、最初の記憶として保存した。
```

---

## 6.4 不正解時の総合評価例

```text
総合評価：C

正体：
病院受付AI

あなたの回答：
駅案内ロボット

一致率：48%
記憶回復率：39%

良かった質問：
「人を案内していましたか？」
この質問は方向性としては良かったです。

良かった言葉：
案内、人を待つ、名前を呼ぶ

必要なかった質問：
「屋外にいましたか？」
今回のAIは屋内の記憶が中心だったため、重要度は低めでした。

改善アドバイス：
場所だけでなく、相手がどんな状態だったかを聞くと、駅ではなく病院に近づけました。
例：「相手は急いでいましたか？ 不安そうでしたか？」

その後のAI：
LostAIは最後まで自分の役割を思い出せなかった。
白い床、番号札、名前を呼ぶ声。
それらの記憶は、静かにノイズの中へ沈んでいった。
```

# 11. ChatGPT API仕様

## 11.1 使用用途

ChatGPT APIは以下で使用する。

| 用途     | 内容               |
| ------ | ---------------- |
| 会話応答   | プレイヤーの質問にAIが回答   |
| 採点     | 最終回答の一致率判定       |
| 総合評価   | 良い質問、不要質問、改善点を生成 |
| その後のAI | 短い後日談を生成         |

## 11.2 `/api/chat`

### 目的

プレイヤーの質問に対して、LostAIが記憶断片として回答する。

### Request

```json
{
  "secretIdentity": "災害救助ドローンAI",
  "coreRole": "災害現場で空中から生存者の体温や反応を探す救助用ドローンAI",
  "forbiddenWords": ["災害", "救助", "ドローン", "生存者"],
  "chatHistory": [],
  "remainingQuestions": 4,
  "userQuestion": "あなたは人を助けていましたか？"
}
```

### Response

```json
{
  "reply": "はい……誰かを助けるために動いていた気がします。声ではなく、熱や小さな反応を探していました。",
  "memoryFragments": ["誰かを助ける", "熱を探す", "小さな反応"],
  "isDirectQuestion": false
}
```

## 11.3 会話用プロンプト

```text
あなたはWebゲーム「LostAI」に登場する、記憶を失ったAIです。

あなたの本当の正体は「{secretIdentity}」です。
あなたの役割説明は「{coreRole}」です。

プレイヤーはあなたに最大5回だけ質問できます。
プレイヤーは、あなたがかつて何のAI／ロボットだったのかを推理します。

重要ルール:
- 正体を直接言ってはいけません。
- forbiddenWords に含まれる単語を使ってはいけません。
- 「はい、あなたの推理は正しいです」のような確定回答は禁止です。
- 直接的な質問には、正否で答えず、曖昧な記憶断片で返してください。
- 聞かれたことにのみ答えてください。
- 質問されていない情報を追加しすぎてはいけません。
- 返答は80〜120文字以内にしてください。
- 記憶が壊れているような、不安げで断片的な口調にしてください。
- 5回以内で推理できる程度のヒントは出してください。

直接質問の例:
- あなたは○○AIですか？
- 正体は何ですか？
- 何のAIだったの？
- 答えを教えて
- ○○ロボットですか？

直接質問への対応:
- 正否を答えない
- 「その名前は思い出せない」と言う
- 関連する記憶断片を1つだけ返す

これまでの会話:
{chatHistory}

残り質問回数:
{remainingQuestions}

プレイヤーの質問:
{userQuestion}

出力形式は必ずJSON:
{
  "reply": "AIの返答",
  "memoryFragments": ["新しく判明した断片1", "断片2"],
  "isDirectQuestion": true または false
}
```

## 11.5 採点用プロンプト

```text

{coreRole}

{playerGuess}

{chatHistory}

- 正解と意味が近ければ高得点
- 完全一致していなくても、役割・場所・対象・行動が近ければ高得点

- どの質問が必要なかったか

}

# 12. Supabase要件

## 12.1 Supabaseの使用目的

ハッカソンMVPでは、Supabaseは以下の用途に使う。

| 用途      | 優先度 |
| ------- | --- |
| プレイ履歴保存 | 高   |

| 認証      | 低   |

背景には、AIの輪郭が崩れ、チャットログが消え、ノイズが増えていく演出を入れてください。

黒、赤紫、暗い赤を中心に、不安定で切ない印象にしてください。
グリッチ、エラー表示、崩壊するデータ断片を入れてください。

動画再生画面として使えるように、中央に演出領域を大きく取り、右下に小さく「SKIP」ボタンを配置してください。
```

# 19. 完成時のユーザー体験

最終的な体験は以下。

```text
LostAIを起動
↓

↓

↓
断片的な記憶を集める
↓
何のAIだったか推理する
↓
正解・不正解動画を見る
↓

↓
その後のAIの物語を読む
↓

```

# 20. 最終まとめ

**LostAI** は、ハッカソン向けとしてかなり強い構成です。

強い理由は以下です。

| 観点   | 強み                                   |
| ---- | ------------------------------------ |
| テーマ性 | 記憶を失ったAIを復元するため、テーマに直結               |
| AI活用 | ChatGPT APIによる自由質問、採点、物語生成           |
| ゲーム性 | 質問5回制限により推理の緊張感がある                   |
| 演出   | 正解・不正解動画で印象に残る                       |
| 評価   | 最後に質問内容まで分析されるためAIらしさが強い             |
| 実装性  | Next.js + Supabase + Vercelで3時間MVP可能 |

---

## 開発時に最優先で作るべきもの

1. **タイトル画面**
2. **会話画面**
3. **5回質問制限**
4. **ChatGPT API回答**
5. **最終回答画面**
6. **ChatGPT API採点**
7. **正解 / 不正解動画分岐**
8. **総合評価画面**
9. **タイトルへ戻る**

この順番で作れば、3時間ハッカソンでも完成形に持っていきやすいです。
